## 🌐 Deploy Frontend

### Option 1: Vercel (RECOMMENDED - Easiest)

```bash
cd frontend

# Install Vercel CLI
npm install -g vercel

# Deploy
vercel

# Follow prompts:
# - Link to existing project? No
# - Set project name: algovision
# - Framework: Vite
# - Build command: npm run build
# - Output directory: dist
```

**Result:** Your frontend is live! Get the URL from Vercel dashboard.

### Option 2: Netlify

```bash
cd frontend
npm run build

# Visit netlify.com
# Drag and drop dist/ folder to Netlify
# Get your live URL
```

### Option 3: GitHub Pages

```bash
cd frontend
npm run build

# Push dist/ folder to gh-pages branch
# Enable GitHub Pages in Settings → Pages → Deploy from branch → gh-pages
```

## 🔌 Deploy Backend

### Option 1: Render (RECOMMENDED - Free)

1. Create account at **https://render.com**
2. Create New → Web Service
3. Connect GitHub repository
4. Configuration:
   - **Name:** algovision-backend
   - **Environment:** Python 3
   - **Build command:** `pip install -r requirements.txt`
   - **Start command:** `python app.py`
   - **Root directory:** backend
5. Create Web Service
6. Get backend URL from Render

### Option 2: Heroku

```bash
# Login to Heroku
heroku login

# Create app
heroku create algovision-backend

# Add Python buildpack
heroku buildpacks:add heroku/python -a algovision-backend

# Push code
git push heroku main

# Get URL
heroku open
```

### Option 3: AWS/DigitalOcean/Linode

Standard Flask deployment process:
1. Create server instance
2. Install Python, pip
3. Clone repository
4. Install dependencies: `pip install -r requirements.txt`
5. Run: `python app.py`
6. Configure domain/DNS

## 🔗 Connect Frontend to Backend

### After Deploying Backend

Update **frontend/.env** (or create it):

```env
VITE_API_URL=https://your-backend-url.com/api
```

For development (local):
```env
VITE_API_URL=http://localhost:5000/api
```

Rebuild frontend:
```bash
cd frontend
npm run build
redeploy to Vercel/Netlify
```
